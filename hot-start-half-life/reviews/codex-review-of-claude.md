# Codex Review of Claude Round 1

I would reject this in its current form. The projection tables are useful descriptive shrinkage, but the report sells false precision and labels methods as more sophisticated than the implementation.

The stabilization headline is the biggest problem. `claude-analysis/REPORT.md:11` and `:17` claim three stats shifted with non-overlapping bootstrap CIs. But `claude-analysis/stabilization.py` never bootstraps player-seasons. It fixes the qualifying player-season set at lines 214-221, then each "draw" only random-splits PAs inside the same players at lines 235-243. Those intervals are random-partition uncertainty, not sampling uncertainty over MLB player-seasons or years, and they ignore repeated-player dependence. That alone invalidates the kill-gate language. The code also extrapolates right-censored curves with a line through the last four points (`stabilization.py:177-190`) and caps remaining failures (`:247-250`). Maybe the point estimates are suggestive, but FanGraphs should not publish "the shift is real" (`REPORT.md:74`) from this CI.

Also, 2022-2025 versus Carleton constants is not the same as "the 2026 rule environment." `REPORT.md:70` says ABS challenges have not shifted BB/K, but the stabilization corpus excludes 2026. My counterfactual asked a different but relevant question: current-era versus broad-era predictions on the 2026 hitters gave average delta -0.0012 wOBA, 95% CI [-0.0196, 0.0152] (`codex-analysis/findings.json:2-7`). These can coexist only if Claude's shift is historical-definition drift, not evidence that 2026 hot-start half-life changed.

The Bayesian projection section is mislabeled. `claude-analysis/bayes_projections.py:9-15` promises a hierarchical model with `kappa ~ HalfNormal`; the actual model is one fixed-prior `rho ~ Beta(alpha,beta)` with Binomial likelihood (`:261-264`). Posterior alpha/beta are computed directly (`:289-290`), so NUTS/R-hat theater adds nothing. Worse, effective prior PA is the point estimate from the flawed stabilization summary (`:157-167`), treated as known. No uncertainty in the stabilization estimate enters the player posteriors. For wOBA, they round continuous weighted wOBA into Bernoulli successes (`:395-400`); Pages becomes 38/94 (`data/bayes_projections.json:127-146`). That variance model is made up. The q10-q90 intervals look tight partly because 489 "prior trials" are injected for wOBA.

Murakami is not reproducible. `REPORT.md:28` says MLB Stats API resolved him to 808959, but `data_pull.py` only uses `playerid_lookup` (`:172-193`), and the log says `[lookup-empty] Munetaka Murakami` (`claude-analysis/logs/data_pull.log:50-54`). The parquet was manually edited or cached. If `data/named_hot_starters.parquet` is absent, `analyze.py` will not recreate the headline Murakami SIGNAL case. And the SIGNAL verdict is mostly an artifact of a deliberately weak league-average 60 PA prior (`bayes_projections.py:225-242`). `REPORT.md:131-143` admits there is no NPB translation, then still calls him the "cleanest" real finding. No. With the exact missing prior that matters most, the honest label is unresolved or ambiguous.

Pages NOISE is directionally plausible but under-modeled. The brief explicitly names 95th-percentile hard-hit rate and asks for EV/HardHit/Barrel/xwOBA features (`RESEARCH_BRIEF.md:55,77-78`). Claude's projection loader reads only event outcome fields (`bayes_projections.py:607-618`), and `player_22g_window_2026` returns only BB/K/BABIP/ISO/wOBA/HR (`:129-146`). So when `REPORT.md:111` says the hot start is "fully explained" by BABIP, the model literally cannot see the contact-quality evidence that might falsify that claim. My QRF also calls Pages noise, but it does so while using EV p90, xwOBA, and barrel features. Claude's conclusion is a blind guess that happens to land in the same place.

Miller's streak model should be deleted. `REPORT.md:170-181` publishes 65%/45%/33% survival probabilities. The code models first ER as geometric on a Beta-mixed ER/BF rate (`bayes_projections.py:532-538`), but the "observed ER" input is actually observed HR (`:529-531`). Non-HR earned runs are a failure mode; baserunner sequencing is the main failure mode for a scoreless streak. The comment at lines 524-525 mentions baserunner density, but the code never uses the five baserunners allowed. It also ignores opponents and leverage. My reliever QRF is admittedly a rough RA9 proxy, but its median expected IP to next run is 2.2 (`codex-analysis/tables/qrf_pitcher_intervals.csv:2`), not 8.6. Claude's published survival percentages are numerology.

The PELT result is also oversold. `REPORT.md:13` converts "no detected breakpoint" into "statistical excursions, not regime changes." But `changepoint.py` only tests per-PA wOBA contribution (`:245`), not the component stats that drive Rice, Miller, or Murakami. Its own docstring promises Murakami will be "insufficient pre-2026 data" (`:8-10`), yet `data/changepoints.json:210-260` gives him "no_break_in_2026" with `first_2026_idx: 0`. That is not a null result; it is an untestable case mislabeled as evidence.

Compared with my pass, the real overlap is that Pages/Rice/Trout all regress hard and the noise floor is brutal. The disagreements are where Claude's machinery is least credible: Murakami relies on an unreproducible weak prior, Miller relies on HR-only streak survival, and the era-effect claim relies on under-bootstrapped split-half curves. I would use Claude's tables as exploratory shrinkage diagnostics, not as publishable claims. The headline "non-null is the story" (`REPORT.md:19`) should be rejected. The safer article remains: April mostly lies, with possible component-level signals for Rice power/discipline and Miller strikeouts, not durable hot starts.
